MnFileInput Demo — Display Modes, Limits & Validation

This demo showcases the MnFileInput component: an accessible file input that implements ControlValueAccessor, renders image previews, and enforces client-side accept / maxSize / maxFiles limits. The control value is a plain File | null (single) or File[] (multiple).

Example 1: Dropzone (default), Single Image, Required

Default dropzone layout with a required validator and a 2 MB maxSize. Pick a non-image or oversized file to see the inline error.

Example 2: Thumbnail Grid, Multiple (max 4)

displayMode: 'thumbnail' with multiple: true and maxFiles: 4. Selected images render as tiles with a remove action.

Example 3: List Mode, Documents

displayMode: 'list' with accept: '.pdf,.doc,.docx,.txt'. Non-image files show a file icon, name, and size.

Example 4: Compact Inline Picker

displayMode: 'compact' renders an inline button plus the current filename.

Example 5: Existing Image via currentUrl

Previews an already-saved image. Removing it emits cleared and leaves the form value untouched (use this to flag a delete on the server).
