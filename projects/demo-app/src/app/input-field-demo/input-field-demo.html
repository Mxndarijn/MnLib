MnInputField Demo - Error Handling & Display Modes

This demo showcases the refactored MnInputField component with English documentation, flexible error handling, and the new showAllErrors feature.

Example 1: Default Built-in Error Messages

Uses built-in English error messages (translated from Dutch). Shows only one error at a time (default behavior).

Example 2: Custom Error Messages Only

useBuiltInErrorMessages: false - Only uses custom error messages. Demonstrates complete control over error text.

Example 3: Priority-Based Error Display (Single Error)

Uses errorPriority to control which error shows first. Custom validator (noXAllowed) + function-based error messages. Try entering 'x' or less than 3 characters.

Example 4: Show All Errors Mode NEW

showAllErrors: true - Displays ALL validation errors simultaneously. Perfect for complex validation rules where users need to see all requirements at once. Try entering a weak password to see multiple errors appear.
